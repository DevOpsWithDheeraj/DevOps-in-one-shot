A **Pod** is the **smallest and simplest deployable unit** in Kubernetes.
It represents **a single instance of a running process** in your cluster.

Each Pod contains **one or more containers** (usually Docker containers) that **share**:

* The **same network namespace** (same IP address and port space)
* The **same storage volumes**
* The **same lifecycle**

## ğŸ§  **Why Do We Need Pods?**

Containers often need to **work together**.
For example:

* One container runs an application (e.g., `nginx`)
* Another container collects logs or sends metrics for that app

You can group these containers into **a single Pod** so they:

* Communicate using `localhost`
* Start/stop together
* Share data via volumes


Here:

* Both containers share `/var/log/nginx` via a volume.
* The `log-agent` reads logs generated by Nginx in real-time.

## ğŸ§  **Pod Lifecycle States**

A Pod can be in these states:

| State                | Meaning                                      |
| -------------------- | -------------------------------------------- |
| **Pending**          | Pod accepted by cluster but not yet running. |
| **Running**          | At least one container is running.           |
| **Succeeded**        | All containers exited successfully.          |
| **Failed**           | At least one container failed.               |
| **CrashLoopBackOff** | Container keeps crashing and restarting.     |

Check status:

```bash
kubectl get pods
kubectl describe pod <pod-name>
```
